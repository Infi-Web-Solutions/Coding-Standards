Need Help With a Standard?

Use this page when a requirement is unclear, a standard is missing, a checklist is confusing, or a blocker needs to be raised properly.

Unclear Requirement

Ask Before Building

If the task requirement is unclear, do not guess silently. Ask a clear question in the ticket before implementation.

Bug / Blocker

Raise It in the Ticket

Important issues should not stay only in chat. Add the problem, impact, and reproduction steps to the ticket.

Improvement

Suggest Standard Updates

If a pattern is repeated or missing from the standards, propose an update so the team can reuse it.

What to Include When Asking for Help

A good help request should give enough context for someone else to respond without guessing.

Situation What to Include Recommended Template
Requirement is unclear Task link, unclear point, options you considered, and your recommended approach Work Log Template
Bug found Steps to reproduce, expected result, actual result, screenshots/logs, affected area Bug Report Template
API behavior unclear Endpoint, request payload, response received, expected response, user role tested API Documentation Template
Form behavior unclear Fields, validation rules, required states, API dependency, Add/Edit difference Form Documentation Template
Task is complete What changed, testing done, evidence, files touched, known limitations Completion Comment Template

Do Not Do This

Anti-Patterns

Do not say only "not working"
Add exact steps, expected behavior, actual behavior, and evidence.
Do not keep blockers only in chat
If it affects delivery, it must be visible in the ticket.
Do not mark complete without evidence
Done requires self-testing and proof.
Do not guess permission or business rules
Backend is the source of truth. Confirm before implementing.
Before Asking

Check the Existing Standards First

Most answers should already exist in standards, checklists, tasks, or templates. Ask for help only after checking the relevant page.
